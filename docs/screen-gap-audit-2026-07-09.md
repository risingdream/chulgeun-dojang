## 총평

기획 원본은 `data-screen-label` 기준 26개 화면이다. 현재 구현에서 직접 확인되는 화면 라벨은 11개뿐이고, 이 중 실제 기획 라벨과 맞는 것은 7개 정도다.

핵심 갭은 세 가지다.

1. 직원 출퇴근 흐름이 `B1/B2`가 아니라 `2a 기기 기억` 화면으로 구현되어 화면 구조가 섞였다.
2. 키오스크 상태 화면 `A2/A3/A4/A5`와 직원 관리 `D1/D2/D3`가 없다.
3. 위치 위험 `B5`는 기획상 핵심인데, 현재는 위치 없음만 표시하고 반경 밖 판단이 없다.

## 코드에서 확인된 추가 문제

- `seedEmployees`가 아직 코드에 남아 있다. 운영 D1에서는 숨겼지만 로컬/메모리 흐름과 테스트에는 목업 직원이 계속 쓰인다.
- `A5` 라벨이 기획 원본과 다르게 쓰인다. 원본 `A5`는 폰 오프라인인데, 현재 `A5 키오스크 로그인 PIN`이다.
- 직원 흐름 라벨이 잘못 붙어 있다. `B1/B2`가 화면 이름으로 살아야 나중에 테스트와 기획 비교가 쉬워진다.
- 위치 반경 계산이 없다. `workspaces.latitude/longitude/radius_meters`를 저장하지만 `buildRiskFlags`에서 쓰지 않는다.
- 관리자 화면은 `오늘 기록`만 있고, 직원 관리 진입점이 없다.

## 우선순위

### P0. 바로 고쳐야 함

1. `D1/D2/D3` 직원 관리 화면 구현
2. `B1/B2` 직원 흐름을 원본 라벨과 화면 구조로 분리
3. `B5` 위치 반경 밖 판단 구현
4. `seedEmployees` 완전 제거 또는 테스트 전용 fixture로 격리
5. `A5` 라벨 충돌 수정

### P1. 키오스크 완성도

1. `A2` 오프라인 화면
2. `A3` 큐알 갱신 실패 화면
3. `A4/A5` 폰 키오스크 전용 레이아웃
4. A7 자동 잠금 카운트와 큐알 유지 표현 강화

### P2. 다음 버전

1. `C1~C5` 패스키 흐름
2. 출근 번호 `2b`
3. 출근 카드 `2c`

## 추천 작업 순서

1. 먼저 직원 관리 `D1/D2/D3`부터 붙인다. 지금은 실제 직원을 넣을 방법이 없어서 제품 사용이 막힌다.
2. 그 다음 직원 스캔 흐름을 `B1 → B2 → B3 → B4/B5`로 라벨과 화면을 다시 맞춘다.
3. 이어서 위치 반경 계산과 `B5` 위험 완료 화면을 만든다.
4. 마지막으로 키오스크 상태 머신 `A2/A3/A4/A5`를 붙인다.

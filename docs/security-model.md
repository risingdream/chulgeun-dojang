# 보안 설계

## 목표

완전한 조작 불가능이 아니라, 조작을 어렵게 만들고 흔적을 남기는 것이 목표다.

## 방어 수단

1. 현장 큐알
   - 키오스크나 사장님 화면에 표시된 짧은 만료 큐알이 필요하다.
2. 패스키
   - 직원 슬롯에 등록된 공개키로만 출퇴근 서명을 검증한다.
3. 위치 확인
   - 사업장 반경 안인지 확인하고, 없거나 멀면 위험 기록을 남긴다.
4. 추가 전용 기록
   - 출퇴근과 정정은 지우거나 덮어쓰지 않고 이벤트로만 쌓는다.
5. 해시 체인
   - 각 근태 이벤트는 이전 이벤트 해시를 참조해 나중에 변조 흔적을 알 수 있게 한다.

## 큐알 재사용 정책

기본 정책은 전역 1회 사용이다.

- 같은 큐알 해시는 사업장 안에서 한 번만 성공할 수 있다.
- 첫 성공 요청은 `qr_consumptions`에 `qr_nonce_hash`를 원자적으로 넣고 통과한다.
- 이미 소비된 큐알이면 다른 직원, 같은 직원 모두 거절한다.
- 키오스크는 성공 응답이나 사용된 큐알 응답을 받으면 즉시 새 큐알을 표시한다.
- 여러 직원이 같은 시간대에 출근할 때는 같은 큐알 공유가 아니라 빠른 큐알 재발급으로 처리한다.

## 왜 직원별 1회 사용이 아닌가

직원별 1회 사용은 운영은 편하지만, 큐알 사진 공유를 막는 힘이 약하다. 무료 근태관리의 핵심 가치가 조작 가능성 낮추기라면 전역 1회 사용이 기본값이어야 한다.

# 보안 설계

## 목표

완전한 조작 불가능이 아니라, 조작을 어렵게 만들고 흔적을 남기는 것이 목표다.

## 방어 수단

1. 현장 큐알
   - 키오스크나 사장님 화면에 표시된 짧은 만료 큐알이 필요하다.
2. 패스키
   - 직원 슬롯에 등록된 공개키로만 출퇴근 서명을 검증한다.
3. 위치 확인
   - 사업장 반경 안인지 확인하고, 없거나 멀면 위험 기록을 남긴다.
4. 추가 전용 기록
   - 출퇴근과 정정은 지우거나 덮어쓰지 않고 이벤트로만 쌓는다.
5. 해시 체인
   - 각 근태 이벤트는 이전 이벤트 해시를 참조해 나중에 변조 흔적을 알 수 있게 한다.

## 큐알 재사용 정책

기본 정책은 직원별 1회 사용이다.

- 같은 직원, 같은 출퇴근 종류, 같은 큐알 해시는 한 번만 허용한다.
- 키오스크는 성공 응답을 받으면 즉시 새 큐알을 표시한다.
- 여러 직원이 같은 시간대에 출근할 수 있으므로 큐알 전역 1회 사용은 선택 기능으로 둔다.

## 전역 1회 사용을 켤 때

`qr_consumptions` 테이블에 `qr_nonce_hash`를 유일값으로 넣고, 성공한 첫 요청만 통과시킨다. 이 방식은 더 강하지만 동시에 찍는 직원 경험이 나빠진다.

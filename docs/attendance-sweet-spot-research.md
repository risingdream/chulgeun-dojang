# 근태 편의성 스윗스팟 리서치

## 결론

출근도장의 스윗스팟은 `공용 키오스크 큐알 + 직원 개인폰 패스키 + 위치 소프트 체크`다.

앱 설치, 얼굴 인식, 비콘 장비, 전용 태블릿 필수화를 피하면서도 대리 출근과 큐알 사진 공유를 막는 힘을 유지한다.

## 방식별 판단

### 1. PIN 키오스크

- 장점: 가장 쉽고 빠르다.
- 단점: 번호 공유와 대리 출근에 약하다.
- 판단: 기본값으로 두면 제품의 차별점이 사라진다.

### 2. 큐알 + 개인폰

- 장점: 앱 없이 가능하고 현장에 있어야 한다.
- 단점: 큐알 사진 공유와 동시 출근 충돌을 설계로 막아야 한다.
- 판단: 출근도장의 기본 축으로 적합하다.

### 3. 큐알 + 패스키

- 장점: 생체 인증으로 직원 기기를 묶을 수 있고 비밀번호가 없다.
- 단점: 초기 등록 한 번은 필요하다.
- 판단: 편의성과 조작 방지의 균형이 가장 좋다.

### 4. 위치/GPS

- 장점: 현장성 보강에 좋다.
- 단점: 브라우저 권한, 실내 오차, 개인정보 부담이 있다.
- 판단: 기본 차단 조건보다 위험 기록으로 먼저 쓰는 편이 낫다.

### 5. 얼굴/셀피

- 장점: 대리 출근 방지력이 강하다.
- 단점: 개인정보 부담, 저장 비용, 거부감이 크다.
- 판단: 무료 오픈소스 MVP 기본값으로는 과하다. 고위험 사업장 선택 옵션으로만 둔다.

### 6. WiFi, 비콘, NFC

- 장점: 현장 인증은 강하고 직원 행동은 단순하다.
- 단점: 장비, 앱 권한, 설치 안내가 필요하다.
- 판단: 나중에 유료/고급 모드로 적합하다. MVP 기본값은 아니다.

## 권장 기본 흐름

### 직원 등록

1. 사장님이 직원 슬롯을 만든다.
2. 직원이 일회용 등록 링크를 연다.
3. 이름 확인 후 패스키를 등록한다.
4. 앱 설치와 비밀번호는 없다.

### 매일 출퇴근

1. 사업장 키오스크 화면에 큐알이 뜬다.
2. 직원이 개인폰 카메라로 큐알을 찍는다.
3. 브라우저가 열리고 패스키 인증을 한다.
4. 위치 정보가 있으면 함께 남긴다.
5. 성공 화면은 짧게 보여주고 끝낸다.

## 큐알 정책 보완

전역 1회 사용 정책은 유지하되, `성공 시 소비`보다 `첫 스캔 시 소비`가 편의성에 유리하다.

- 큐알은 첫 스캔 요청에서 원자적으로 소비된다.
- 첫 스캔자는 `attempt_id`를 받고 패스키 인증을 이어간다.
- 키오스크는 큐알 소비 이벤트를 감지하면 즉시 다음 큐알을 표시한다.
- 뒤늦게 같은 큐알을 찍은 직원은 `이미 갱신된 큐알이야. 화면의 새 큐알을 찍어줘.`를 본다.
- 출퇴근 이벤트는 `attempt_id + 패스키 서명 + 직원 공개키`가 모두 맞아야 생성된다.

이렇게 하면 여러 직원이 출근 시간에 줄을 서도 한 명이 인증을 끝낼 때까지 기다릴 필요가 없다. 각 직원은 새로 바뀐 큐알을 순서대로 찍고 자기 폰에서 인증하면 된다.

## 기본 정책값

| 항목 | 권장값 |
| --- | --- |
| 큐알 만료 | 30초 |
| 큐알 소비 | 첫 스캔 시 전역 1회 소비 |
| 키오스크 갱신 | 소비 즉시 갱신, 그 외 20~30초 자동 갱신 |
| 인증 | 패스키 필수 |
| 위치 | 기본은 기록, 사업장 설정으로 엄격 차단 가능 |
| PIN | 기본 미제공, 사장님 수동 정정으로 대체 |
| 셀피 | 기본 미제공, 고위험 사업장 선택 옵션 |
| 오프라인 | MVP 제외, 나중에 키오스크 로컬 큐 검토 |

## 제품 포지션

`알밤/시프티처럼 무겁지 않고, Clockify처럼 가볍지만, PIN보다 안전한 무료 웹 근태 기록기`.

핵심 차별점은 세 가지다.

1. 직원 앱 설치 없음
2. 사장님 초기 설정 5분 이하
3. 패스키와 큐알 전역 1회 사용으로 대리 출근 흔적을 남김

## MVP에 넣을 것

- 키오스크 큐알 화면
- 직원 등록 링크
- 패스키 등록/인증
- 큐알 첫 스캔 소비 방식
- 위치 소프트 체크
- 추가 전용 이벤트 로그
- 월별 근태표와 CSV
- 사장님 수동 정정 이벤트

## MVP에서 빼야 할 것

- 얼굴 인식
- 셀피 저장
- 비콘/NFC 장비
- 복잡한 근무제도 설정
- 급여 계산
- 전자계약
- 알림톡

## 한 문장 원칙

직원은 `카메라로 찍고 생체 인증만 한다`, 사장님은 `기록과 이상 흔적만 본다`.

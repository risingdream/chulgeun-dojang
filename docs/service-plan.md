# 출근도장 서비스 기획서

작성일: 2026-07-09

## 1. 한 줄 정의

출근도장은 소규모 사업장 사장님이 직원 앱 설치 없이 큐알로 출퇴근을 기록하고, 월말 급여 준비에 바로 쓸 수 있는 가장 가벼운 근태 장부다.

## 2. 왜 지금 필요한가

소규모 사업장의 근태 관리는 아직도 종이, 카톡, 엑셀, 기억에 많이 기대고 있다. 기존 근태 앱은 기능은 많지만 직원 앱 설치, 관리자 설정, 요금제, 장비 설치가 부담이다.

출근도장은 이 틈을 노린다.

- 사장님은 남는 폰이나 태블릿으로 키오스크를 켠다.
- 직원은 자기 폰 카메라로 큐알을 찍는다.
- 기록은 자동으로 쌓이고, 월말에는 CSV로 내려받는다.
- 나중에는 3분월급으로 이어져 급여명세서와 급여대장까지 만든다.

핵심은 “근태 관리 도구”가 아니라 “매일 찍는 출퇴근 습관”을 만드는 것이다.

## 3. 목표 고객

### 1차 고객

직원 1명에서 15명 정도의 소규모 사업장 사장님.

좋은 초기 고객군은 아래 업종이다.

- 카페
- 음식점
- 미용실
- 학원
- 헬스장
- 작은 병원과 의원
- 편의점
- 소형 매장
- 단기 알바를 자주 쓰는 사업장

### 고객이 겪는 문제

- 직원 출퇴근 시간이 말로만 남는다.
- 월말에 급여 계산할 때 기억과 카톡을 뒤진다.
- 직원이 늦게 왔는지, 일찍 갔는지 확인하기 어렵다.
- 근태 앱은 직원에게 설치시키기 귀찮다.
- 유료 근태 서비스는 작은 가게에는 무겁고 비싸게 느껴진다.
- 나중에 다툼이 생기면 남길 자료가 부족하다.

### 구매자는 누구인가

사장님이다. 직원은 구매자가 아니라 참여자다. 그래서 직원 쪽 마찰을 거의 없애야 한다.

직원이 앱을 설치하거나 계정을 만들게 하면 첫 사용 전환이 크게 떨어진다. 직원은 큐알을 찍고, 첫 사용 때 이름을 고른 뒤 다음부터는 자기 폰 기억으로 바로 출퇴근만 누르는 흐름이 맞다.

## 4. 포지셔닝

출근도장은 시프티, 뉴플로이, 이카운트 같은 근태 서비스와 정면으로 싸우면 안 된다. 그쪽은 더 큰 회사와 더 복잡한 근무 제도에 강하다.

출근도장의 자리는 다르다.

> “앱 설치 없는 무료 큐알 출퇴근 장부. 월말에는 3분월급으로 급여 준비까지.”

### 이겨야 하는 상대

진짜 경쟁자는 기존 SaaS가 아니라 아래 습관이다.

- 종이에 적기
- 카톡으로 말하기
- 사장님 기억에 맡기기
- 엑셀에 나중에 몰아서 쓰기
- 급여일 직전에 직원에게 다시 물어보기

출근도장은 이 습관을 바꿔야 한다.

## 5. 서비스 원칙

### 원칙 1. 직원 앱 설치 없음

직원에게 앱 설치를 시키는 순간 작은 사업장은 이탈한다. 출근도장은 웹 링크와 큐알만으로 동작해야 한다.

### 원칙 2. 사장님 기기만 있으면 시작

전용 장비를 팔지 않는다. 남는 폰, 태블릿, 컴퓨터가 키오스크가 된다.

### 원칙 3. 완벽한 조작 방지가 아니라 조작 흔적

소규모 사업장에서 완전한 조작 방지는 과하다. 대신 조작을 어렵게 하고, 위치 없음, 거리 벗어남, 큐알 재사용 같은 흔적을 남긴다.

### 원칙 4. 기록은 지우지 않고 정정한다

출퇴근 기록은 덮어쓰지 않는다. 틀린 기록은 정정 이벤트로 남긴다. 나중에 다툼이 생겼을 때 장부가 된다.

### 원칙 5. 급여 준비로 이어져야 돈이 된다

근태 기록만으로는 월 1만원대가 한계다. 진짜 유료 전환은 월별 마감, 급여명세서, 급여대장, 3분월급 연결에서 나온다.

## 6. 핵심 사용 흐름

### 첫 설정

1. 사장님이 출근도장에 들어온다.
2. 사업장 이름을 입력한다.
3. 직원 이름을 추가한다.
4. 키오스크 화면을 연다.
5. 키오스크를 매장 입구나 카운터에 둔다.

### 매일 출근

1. 직원이 키오스크 화면의 큐알을 자기 폰으로 찍는다.
2. 첫 사용이면 직원이 자기 이름을 선택하고 “이 폰 기억하기”를 켠다.
3. 다음 사용부터는 기억된 직원 이름으로 바로 출근 또는 퇴근을 누른다.
4. 위치 권한을 허용하면 기록 순간의 위치 1회만 저장된다.
5. 위치를 건너뛰면 기록은 막지 않고 위치 없음 흔적을 남긴다.
6. 기록 완료 화면이 뜬다.
7. 같은 큐알을 다시 쓰면 막힌다.

### 월말

1. 사장님이 최근 기록 또는 월별 기록을 확인한다.
2. 이상 기록을 본다.
3. 필요한 정정을 남긴다.
4. CSV를 내려받는다.
5. 다음 단계에서 3분월급으로 급여명세서를 만든다.

## 7. 현재 구현 상태

운영 주소: https://chulgeun-dojang.risingdream.workers.dev

현재 구현된 기능은 아래와 같다.

- Cloudflare Workers 운영 배포
- D1 기반 근태 원장
- `/kiosk` 운영 키오스크
- 관리형 웹서비스용 로컬 사업장 토큰과 키오스크 PIN 로그인
- 짧은 만료 시간을 가진 서명 큐알
- 큐알 첫 스캔 시 전역 1회 소비
- `/scan` 직원 기록 화면
- `/api/clock` 출퇴근 저장
- 직원 폰 기억 쿠키 기반 이름 선택 생략
- 위치 1회 저장 안내와 위치 건너뜀 흔적
- `/admin/today` 사장님 오늘 기록 화면
- `/events` 최근 기록 보기
- `/admin/export.csv` 관리자 토큰 기반 CSV 내려받기
- 미인증 CSV 접근 차단
- 큐알 재사용 차단

아직 구현되지 않은 핵심 기능은 아래와 같다.

- 사장님 계정 로그인
- 직원 추가, 수정, 비활성화 화면
- 강한 직원 개인 기기 등록
- 패스키 기반 직원 확인
- 월별 근태 마감 화면
- 정정 이벤트 화면
- 3분월급 연동

## 9. 화면 설계

### 홈

목표: 사장님이 10초 안에 이해하고 시작하게 만든다.

필수 문구:

- 앱 설치 없는 큐알 출퇴근
- 무료로 시작
- 월말 CSV 내려받기
- 작은 가게용

주요 버튼:

- 무료로 시작하기
- 키오스크 체험하기

### 사업장 설정

필수 입력:

- 사업장 이름
- 기준 위치
- 허용 반경
- 키오스크 이름

처음에는 주소 검색보다 “현재 위치로 설정”이 더 빠르다.

### 직원 관리

필수 기능:

- 직원 이름 추가
- 직원 상태 변경
- 직원별 등록 링크 복사
- 직원 삭제 대신 비활성화

직원 번호, 휴대폰 번호, 주민번호는 첫 버전에서 받지 않는다. 개인정보 부담을 낮춰야 한다.

### 키오스크

필수 요소:

- 큰 큐알
- 남은 시간
- 새 큐알 받기
- 사장님 확인 진입 버튼
- 전체 화면 모드 안내

키오스크 화면은 매장에 계속 켜져 있어야 하므로 글자가 커야 한다. 공용 화면에는 최근 기록과 직원 정보가 보이면 안 된다.

### 직원 출퇴근 화면

필수 요소:

- 첫 사용 때 직원 이름 선택
- 이 폰 기억하기
- 기억된 직원 이름 표시
- 출근 버튼
- 퇴근 버튼
- 위치 1회 저장 안내
- 위치 없이 기록 버튼
- 기록 완료 화면

초기 버전은 이름 선택과 폰 기억 방식으로 시작하고, 다음 버전에서 패스키로 직원 확인을 강화한다.

### 기록 화면

필수 요소:

- 날짜 선택
- 직원별 기록
- 출근 시간
- 퇴근 시간
- 위치 위험 표시
- 큐알 재사용 차단 내역
- CSV 내려받기

## 10. 데이터 정책

### 저장하는 것

- 사업장 이름
- 직원 이름
- 키오스크 이름
- 출퇴근 시각
- 출근 또는 퇴근 유형
- 위치 정보 또는 위치 없음 표시
- 위험 표시
- 큐알 해시
- 정정 이벤트

### 저장하지 않는 것

- 큐알 원문
- 주민번호
- 원문 휴대폰 번호
- 불필요한 위치 이동 기록
- 직원 사생활 기록

### 기록 원칙

출퇴근 기록은 지우거나 덮어쓰지 않는다. 사장님이 고치는 경우에도 원 기록과 정정 이유를 함께 남긴다.

## 11. 조작 방지 정책

### 기본 방어

- 큐알은 짧게 만료된다.
- 큐알은 첫 스캔 때 바로 소비된다.
- 이미 소비된 큐알은 다시 쓸 수 없다.
- 위치가 없거나 멀면 위험 표시를 남긴다.
- 패스키가 들어오면 직원 개인 기기와 묶는다.

### 왜 전역 1회 큐알인가

직원별 1회 큐알은 편하지만, 큐알 사진 공유를 막는 힘이 약하다. 출근도장의 핵심 값은 “가볍지만 대충 속이기 어렵다”다. 그래서 전역 1회 소비가 맞다.

### 위치 정책

위치가 없다고 무조건 막지 않는다. 처음에는 위험 표시를 남기는 방식이 맞다. 강제 차단은 사업장 설정으로 나중에 켤 수 있게 한다.

## 12. 가격 전략

### 무료

무료 범위는 넓게 둔다.

- 큐알 출퇴근
- 최근 기록
- CSV 내려받기
- 직원 소수 등록

무료가 넓어야 사장님이 부담 없이 키오스크를 매장에 켜둔다.

### 유료

유료는 “기록”이 아니라 “월말 정리 시간 절약”에서 받아야 한다.

유료 기능 후보:

- 직원 수 추가
- 월별 마감
- 정정 이력
- 3분월급 연동
- 급여명세서 반복 생성
- 급여대장
- 월별 백업
- 여러 사업장 관리

권장 가격대:

- 단독 근태: 월 9,900원에서 14,900원
- 3분월급 번들: 월 19,900원에서 29,900원

### 1차 채널

- 세무전문가 고객사 배포
- 3분월급 무료 도구 묶음
- 소규모 사업장용 검색 유입
- 카페, 음식점, 학원 사장님 커뮤니티

### 검색용 문구

- 무료 출퇴근 기록
- 알바 출퇴근 기록
- 큐알 근태 관리
- 직원 앱 설치 없는 근태
- 작은 가게 근태 관리
- 급여 계산 전 근태 정리

### 확산 장치

CSV와 월말 마감 화면에 출근도장 출처를 자연스럽게 남긴다. 사장님이 세무전문가나 직원에게 공유할 때 제품이 같이 퍼져야 한다.

- 월별 마감 사용 수

## 15. 첫 출시 기준

첫 출시는 기능 완성이 아니라 “사장님이 매장에 켜두고 쓸 수 있느냐”로 판단한다.

출시 기준:

- 사장님이 3분 안에 사업장을 만들 수 있다.
- 직원 3명을 1분 안에 추가할 수 있다.
- 키오스크가 끊기지 않고 새 큐알을 보여준다.
- 직원이 앱 설치 없이 출퇴근을 기록한다.
- 월말에 CSV를 내려받을 수 있다.
- 같은 큐알 재사용은 막힌다.
- 위험 기록이 눈에 보인다.

- 키오스크 생성

### 2단계: 월별 장부

- 월별 기록 화면
- 직원별 일자 묶음
- CSV 개선
- 정정 이벤트

### 3단계: 직원 확인 강화

- 직원 등록 링크
- 패스키 등록
- 직원 기기 바인딩
- 패스키 서명 검증

### 4단계: 3분월급 연결

- 월별 마감 데이터 만들기
- 급여 계산 입력값으로 넘기기
- 급여명세서 반복 생성 흐름 연결

### 5단계: 유료화

- 무료 한도 설정
- 결제
- 여러 사업장 관리
- 백업과 보관

## 17. 의사결정 메모

출근도장은 단독으로 큰 SaaS가 되기보다 3분월급의 앞단 진입 상품으로 보는 게 맞다. 작은 사업장은 근태만 따로 돈을 내기 어렵다. 하지만 근태가 급여 준비와 연결되면 돈을 낼 이유가 생긴다.

그래서 제품의 중심 문장은 아래가 좋다.

> “매일은 큐알로 찍고, 월말에는 급여 준비까지.”

## 18. 바로 해야 할 일

1. 홈 문구를 “무료 큐알 출퇴근 장부” 중심으로 바꾼다.
2. 사장님 로그인과 사업장 생성부터 만든다.
3. 직원 관리 화면을 만든다.
4. 월별 근태표와 CSV를 상품의 핵심 화면으로 만든다.
5. 3분월급으로 넘길 데이터 형식을 정한다.

## 19. 한 줄 결론

출근도장은 근태 SaaS가 아니라 소규모 사업장의 월말 급여 준비를 시작하게 만드는 초경량 출퇴근 장부다.
